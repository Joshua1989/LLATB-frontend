Before You Use Team Strength Calculator

If you have used LLSIF-AutoTeamBuilder before, our SIS owning information will be load automatically, otherwise all SIS numbers are set to 9.

Besides, if this is the first time you use the team strength calculator, a default team will be given, you can edit it afterwards.

Directly Import Team

Import from School Idol Tomodachi Account

If you have an account at School Idol Tomodachi, then you can directly import your account.

Note: all cards imported are max level and max bond.

Import from Screenshot

To enable this function, please visit LLATB using HTTP instead of HTTPS.

Thanks to liaocm, a much fancier feature now has been incorporated into LLATB, you can even import a team by screen shot! Due to the server load and the fact that image has to be uploaded, sometimes you need to be patient to wait the recognition result to arrive. And with small probability the team member CAN be recognized incorrectly.

One thing you should keep in mind is the screen shot does not contain the information of skill level and skill slot number, by default all imported cards are max level and max bond, with skill level 1 and slot number same as normal idolization (if a card is unidolized, the slot number is the minimum possible number, if a card is idolized and is not promo, increase by one). So perhaps you still need to manually edit a bit.

Manually Edit Team

Choose Card

The user interface of card filter (idol member, attribute, idolization) is same as profile editing page in LLATB, so I will skip it here. You can read more above it here.

The only different part is that: when you click an icon, nothing will change in your team, the selected card is just highlighted to indicate you that this is the card you will use for card replacement. As you can tell from the card edit modal, the "Remove" button is replaced by the "Replace" button.

Edit SIS Owning Info

If you want to use the auto SIS allocation function, the SIS owning information is necessary. By default the SIS owning information is loaded from your user profile if you have one, the only thing worthing being aware of is that the changes you made here WILL OVERWRITE the SIS owning information in your user profile.

Equip SIS

You can simply equip SIS by checking the check boxes. From the picture below you can see for different SIS types, I did not specify the attribute:

  • For SIS type that is not charm, heal and trick: the SIS has no constraint on the attribute, so the SIS has same attribute as the center card.
  • For SIS type that is charm, heal or trick: the SIS has constraint on the attribute, so the SIS has same attribute as the card equipping it.

If the total cost for equipped SIS exceed the card slot number, the header will turn red to indicate you are violating the constraints.

For example, in the picture below, Umi is equipped with a Perfume and a Heal, since the center card is Smile, so in this case the equipped SISs are Smile Perfume and Empress Heal.

Swap Position of Cards

To swap to cards in the team, simply click on the header cell, you will see the font color becomes green, it tells you this card is waiting for position swapping. You can click on the second header cell to finish the swap.

Do not forget to SAVE your changes before clicking "Back" for computation.

Compute Team Strength Details

Start the Calculation

Again the user interface is almost same as the AutoTeamBuilder, the only difference is the "Auto Mode" button:

  • If the button is OFF: the computation is exactly based on the team you edited
  • If the button is ON: an auto SIS allocation will be performed, this will give an optimal SIS allocation and positioning based on your input and SIS owning information.

Output for Further Trial

Once you have new team from the auto mode, you may want to start from this one to make further changes. You can easily copy the output team JSON and paste it as input team JSON.

Validate Output by Simulation

This feature is add at July 24, 2017. As long as the live you are optimizing on is not default live (which does not have note list), you can run a simulation to see

  • The distribution of game scoring over 5000 trials.
  • The average score obtained by the base attribute, and skill for each card, the average score obtained by Trick SIS for each card.
  • The fraction of each note to be covered by P-lock over 5000 trials.
